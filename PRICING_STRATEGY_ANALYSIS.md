# L-Apply 価格戦略 & 機能制限の考察

## 🎯 モニター価格: ¥980/月

### この価格設定の評価

#### ✅ メリット
1. **心理的障壁が低い**
   - ¥1,000以下は「試してみよう」と思える価格
   - Netflix（¥990）と同価格帯で親近感

2. **競合との差別化**
   - 一般的な予約システム: ¥3,000〜¥10,000/月
   - 圧倒的な価格優位性

3. **口コミが広がりやすい**
   - 「これで980円！？」という驚き
   - 紹介したくなる価格

4. **初期顧客獲得に最適**
   - 100社獲得で月98,000円
   - まずはユーザー数を増やす戦略

#### ⚠️ 注意点
1. **値上げ時の抵抗**
   - モニター → 本価格への移行計画が重要

2. **コスト回収**
   - Firebase Functions、Vercelの費用
   - サポートコスト

3. **ブランドイメージ**
   - 安すぎると「安かろう悪かろう」と思われる可能性

---

## 📊 機能制限の考察

### 結論: **ステップ配信は制限すべき**

理由：
1. ステップ配信は**最も価値の高い機能**
2. コスト的にも負荷が高い（Cloud Functions実行回数）
3. 差別化ポイントとして明確

## 🎯 ステップ配信制限の具体的な実装

**メリット**
- わかりやすい

```

```

**メリット**
- スタンダードでも基本的な自動化が可能
- プロの価値がわかりやすい

```
モニター: なし
スタンダード: 月100通まで
プロ: 無制限
```

**メリット**
- 使った分だけ課金感覚
- 従量課金への移行もスムーズ

## 🎊 結論

### ステップ配信は制限すべき？

**YES！制限すべきです**

**理由**:
1. ✅ 最も価値の高い機能 = 差別化ポイント
2. ✅ アップセルの理由が明確
3. ✅ モニター価格でも十分な価値を提供できる
4. ✅ 「開発中」として期待感を演出
5. ✅ コスト削減（Cloud Functions実行回数）

**具体的な戦略**:
```
モニター（¥980）: ステップ配信なし
↓
「もっと自動化したい！」
↓
スタンダード（¥3,980）: 3ステップ
↓
「条件分岐も使いたい！」
↓
プロ（¥6,980）: 10ステップ + 条件分岐
```
